# 用户与权限管理

## 功能概述

RBAC（基于角色的访问控制）模块，包含用户管理、角色管理、权限管理和菜单管理四个子模块。超管可以创建用户、分配角色、调整权限码、配置侧边栏菜单。

## 操作步骤

### 1. 用户管理

**进入**：侧边栏 → 权限管理 → 用户

#### 新建用户

1. 点击右上角「新建用户」按钮
2. 填写表单：
   - 用户名：登录账号（唯一，不可修改）
   - 邮箱：用于登录和找回密码
   - 显示名：可选，用于展示
   - 密码：至少 6 位
   - 状态：正常 / 禁用
   - 角色：选择一个或多个角色（如 content_admin）
3. 点击「确定」保存

#### 编辑用户

1. 在用户列表中找到目标用户
2. 点击「编辑」按钮
3. 可修改：邮箱、显示名、状态、角色
4. 用户名不可修改
5. 点击「确定」保存

#### 重置密码

1. 点击用户行的「重置密码」按钮
2. 输入新密码（至少 6 位）
3. 点击「确认」

#### 删除用户

1. 点击「删除」按钮
2. 确认弹窗中点击「确认」
3. 超管账号不可删除

#### 批量删除

1. 勾选列表左侧复选框（可多选）
2. 点击顶部出现的「批量删除」按钮
3. 确认后批量删除

### 2. 角色管理

**进入**：侧边栏 → 权限管理 → 角色

#### 新建角色

1. 点击「新建角色」
2. 填写：
   - 角色编码：唯一标识（如 `content_editor`）
   - 角色名称：展示名称（如「内容编辑」）
   - 描述：可选
   - 状态：正常 / 禁用
3. 保存后进入角色详情页分配权限

#### 分配权限

1. 在角色列表中点击角色名称或「编辑」
2. 切换到「权限」标签
3. 勾选该角色需要的权限码
4. 点击「保存权限」

### 3. 权限管理

**进入**：侧边栏 → 权限管理 → 权限

- 展示系统所有权限码的列表
- 可修改权限的「名称」和「描述」（权限编码和资源/动作不可修改）
- 支持按资源类型筛选

### 4. 菜单管理

**进入**：侧边栏 → 权限管理 → 菜单

#### 菜单树操作

- 拖拽节点调整排序
- 点击「编辑」修改菜单名称、路径、图标、权限码
- 点击「删除」移除菜单（有子节点时需先删除子节点）

#### 新增菜单

1. 点击「新增菜单」
2. 填写：
   - 名称：如「数据备份」
   - 路径：如 `/cms/backup`
   - 图标：Icon 名称（如 `ArchiveOutline`）
   - 父菜单：选择父级（为空则为顶级菜单）
   - 权限码：控制谁能看到该菜单
3. 保存后刷新页面生效

## 注意事项

- **超管特权**：`is_super_admin = true` 的用户不受任何权限限制，可看到所有菜单和操作
- **角色禁用**：禁用角色后，拥有该角色的用户将失去对应权限
- **菜单缓存**：菜单树在登录时加载并缓存，修改菜单后需重新登录或刷新页面
- **默认角色**：系统预设 `super_admin`、`content_admin`、`viewer` 三个角色

## 权限要求

| 操作 | 所需权限 |
|---|---|
| 查看用户列表 | `user:list` |
| 新建用户 | `user:create` |
| 编辑用户 | `user:update` |
| 删除用户 | `user:delete` |
| 查看角色 | `role:assign` |
| 分配权限 | `role:assign` |
| 查看权限列表 | `role:assign` |
| 管理菜单 | `menu:manage` |
