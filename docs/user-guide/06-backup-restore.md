# 数据备份与恢复

## 功能概述

将博客全库数据（文章、标签、分类、友链及关联关系）导出为 JSON 文件，或从 JSON 文件还原数据。适用于数据迁移、定期备份、灾难恢复。

## 操作步骤

### 1. 导出数据

**进入**：侧边栏 → 运维 → 备份（或顶部「数据导入导出」）

1. 在「导出全库」卡片中点击「导出 JSON 备份」
2. 浏览器自动下载 `blog-backup-YYYY-MM-DD.json` 文件
3. 文件包含：
   - `version`: 2（版本标识）
   - `exportedAt`: 导出时间
   - `posts`: 所有文章
   - `tags`: 所有标签
   - `categories`: 所有分类
   - `links`: 所有友链
   - `postTags` / `postCategories`: 文章与标签/分类的关联

### 2. 导入数据

1. 在「导入全库」卡片中点击「选择 JSON 备份文件」
2. 选择之前导出的 `.json` 文件
3. 系统显示导入预览（文件内含数据量）
4. 点击「确认导入（覆盖）」
5. 确认弹窗警告：该操作会清空现有数据并覆盖
6. 导入成功后显示结果（成功导入的文章/标签/分类/友链数量）

## 注意事项

- **覆盖式导入**：导入会清空现有数据再写入，操作不可恢复
- **版本兼容**：仅支持 `version: 2` 的备份文件，旧版 v1 备份会被拒绝
- **文件大小**：备份文件可能较大（100~500 篇文章可达数 MB），确保网络稳定
- **敏感信息**：备份文件包含所有文章内容，请妥善保管
- **审计记录**：导出和导入操作会记录在审计日志中

## 权限要求

| 操作 | 所需权限 |
|---|---|
| 导出 | `cms:export` 或 `ops:backup` |
| 导入 | `cms:export` 或 `ops:backup` |

> 该功能通常仅开放给超管，因为导入是破坏性操作。
