# 友链与媒体库管理

## 功能概述

友链管理维护博客的友情链接；媒体库提供图片上传功能，供文章封面和正文引用。

## 操作步骤

### 1. 友链管理

**进入**：侧边栏 → 博客管理 → 友链

#### 新建友链

1. 点击「新建友链」
2. 填写：
   - **标题**：网站名称（如「张三的博客」）
   - **URL**：网站地址（如 `https://example.com`）
   - **图标**：图标 URL，支持相对路径（`/admin-static/...`）或绝对地址
   - **尺寸**：1×1（普通）、2×1（横长）、1×2（纵长）、2×2（大）
   - **排序**：数值越小越靠前
3. 保存

#### 编辑/删除

与文章管理类似，点击对应按钮操作。

#### 图标说明

当前版本图标字段为纯文本输入，可直接粘贴图标 URL。后续版本（Phase 3）将引入图标选择器，支持从 dashboard-icons 库搜索和 favicon 自动获取。

### 2. 媒体库

**进入**：侧边栏 → 博客管理 → 媒体库

#### 上传图片

1. 点击「上传图片」按钮
2. 在弹窗中点击或拖拽图片到上传区域
3. 支持多选，单文件不超过 10MB
4. 上传成功后显示图片预览和 URL

#### 复制图片 URL

1. 上传成功后，图片下方显示完整 URL
2. 点击「复制」按钮复制到剪贴板
3. 在文章编辑页的封面图或正文中粘贴使用

#### 使用限制

- **会话级缓存**：当前版本媒体库仅保存本次会话内上传的图片 URL（刷新页面后从 localStorage 恢复）
- **后端列表**：Phase 3 将补充后端 media 表，实现真正的历史记录管理

## 注意事项

- **图片格式**：支持 JPG、PNG、GIF、WebP
- **图片大小**：单张不超过 10MB（后端限制 5MB）
- **URL 格式**：上传成功后返回 `/admin-static/uploads/xxx.ext` 路径
- **跨域问题**：开发环境通过 Vite 代理 `/admin-static` 到后端，生产环境需确保同域或正确配置 CORS

## 权限要求

| 操作 | 所需权限 |
|---|---|
| 查看友链 | `link:list` |
| 新建友链 | `link:create` |
| 编辑友链 | `link:update` |
| 删除友链 | `link:delete` |
| 上传图片 | `post:list`（媒体库当前复用文章权限） |
